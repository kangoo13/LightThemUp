Pré-requis matériels :

- Coté client :

	- Prise 220V pour branchement de la parure
	- Synthétiseur ou piano pour pouvoir poser la parure et jouer un morceau de musique
	- Appareil ayant une connexion internet (pour télécharger musiques) + connexion bluetooth (connexion a la parure) et capable de lire une application iOS, Android ou Windows Phone (idéal : smartphones, tablettes)

- Coté serveur :

	- Idéalement 2 serveurs linux : un pour l’API + le logiciel de scan de partitions et un pour le front-end du site web (sinon un seul où l’on mettra les deux) Le serveur qui contiendra l’API et le scan devra avoir une bonne bande passante + une bonne performance en RAM et processeurs car il y aura tous les appels (iOS, Android, Windows Phone, Front-end Web)


Ensuite viennent les pré-requis logiciels :

- Coté client :

	- Application (iOS, Android ou Windows Phone) pour pouvoir lancer les musiques à jouer.
	- Optionnel : Un quelconque navigateur web pour acheter des musiques sur notre site web

- Coté serveur :
	
	- Technologie Node.js + Framework express + base de donnée MongoDB pour l’API
	- Fichier jar sur le serveur API permettant de lancer le scan de partitions avec les paramètres adéquats
